## 0. 개요

### 이 문서의 범위
MVP 3대 기능(**테마 초대장 생성 / 이름만 RSVP + 명단 / 링크·OG 공유**)을 **화면 단위로** 구체화한 설계서. 각 화면을 **유저 스토리**에 매핑해, "왜 이 화면이 있는지"부터 컴포넌트·상태·데이터·예외까지 정의한다.

### 대상 유저 2종
- **호스트(H)**: 모임을 여는 사람. 초대장을 만들고 공유하고 현황을 본다.
- **게스트(G)**: 링크를 받은 사람. 가입 없이 열람하고 이름만으로 응답한다.

### 플로우 맵
```
[호스트]  S0 랜딩 ──▶ S1 생성(테마→편집) ──▶ S2 완성·공유 ──┐
                                                        │ 링크 공유(카톡/인스타)
[게스트]                          S3 초대장 열람 ◀───────┘
                                     │
                                     ▼
                                  S4 RSVP 입력 ──▶ S5 응답완료 + 명단(실시간)
                                                        ▲
                                     (호스트도 같은 명단으로 현황 확인) ┘
```

### 라우트 구조 (구현 매핑)
| 화면 | 경로 | 렌더 |
|------|------|------|
| S0 랜딩 | `/` | 정적 |
| S1 생성 | `/create` | 클라이언트 |
| S2 완성·공유 | `/e/[slug]?created=1` | S3와 동일 페이지 + 공유 배너 |
| S3~S5 초대장·RSVP·명단 | `/e/[slug]` | 서버(OG 메타) + 클라이언트(RSVP/명단) |

## 1. 유저 스토리

### 호스트
| ID | 스토리 |
|----|--------|
| **H1** | 호스트로서 **테마를 골라 10초 만에 초대장을 만들고 싶다.** 디자인 고민 없이 감각있게 초대하려고. |
| **H2** | 호스트로서 **날짜·장소가 미정이어도 초대장을 만들고 싶다.** 확정 전에도 미리 사람을 모으려고. |
| **H3** | 호스트로서 **완성 즉시 공유 링크를 받고 싶다.** 바로 카톡방에 뿌리려고. |
| **H4** | 호스트로서 **링크를 붙이면 예쁜 미리보기 카드가 뜨길 원한다.** "센스있다"는 반응을 얻으려고. |
| **H5** | 호스트로서 **누가 오는지 실시간으로 확인하고 싶다.** 몇 명 올지 확신하려고. |

### 게스트
| ID | 스토리 |
|----|--------|
| **G1** | 게스트로서 **링크만 누르면 초대 내용을 보고 싶다.** 가입·앱설치 없이 바로 확인하려고. |
| **G2** | 게스트로서 **이름만 입력해 참석 여부를 남기고 싶다.** 마찰 없이 응답하려고. |
| **G3** | 게스트로서 **누가 오는지 명단을 보고 싶다.** "쟤도 오네? 나도" 하게 되려고. |
| **G4** | 게스트로서 **내 응답을 나중에 바꾸고 싶다.** 계획이 바뀌어도 반영하려고. |

## 2. 화면 명세

### S0. 랜딩 `/`
- **연결 스토리**: H1(진입점)
- **목적**: 제품 한 줄 가치 전달 + 생성 플로우로 유도.
- **구성요소**: 로고/워드마크 · 헤드라인("초대가 곧 기대감이 되는 앱") · 서브카피 · 테마 이모지 4종 미리보기 · `Button`(초대장 만들기)
- **인터랙션**: [초대장 만들기] → `/create`
- **상태**: 단일(정적)
- **데이터**: 없음
- **비고**: 게스트는 이 화면을 거의 안 봄(호스트 진입 전용). 게스트는 S3로 직접 진입.

---

### S1. 초대장 생성 `/create`
- **연결 스토리**: H1, H2
- **목적**: 테마 선택 + 최소 입력으로 초대장 1건 생성.
- **구성요소**
  - 테마 선택: 4종 카드(생일🎂 / 집들이🏠 / 술약속🍻 / 놀러가기✨), 선택 시 강조(ring)
  - `Input` 제목(**필수**)
  - `Switch` 날짜 미정 토글 → 해제 시 `datetime-local`
  - `Switch` 장소 미정 토글 → 해제 시 `Input`
  - `Input` 한 줄 소개(선택)
  - `Button` 초대장 완성
- **인터랙션**
  1. 테마 탭 → 선택 상태 반영
  2. [초대장 완성] → 유효성 검사 → `events` INSERT → `/e/[slug]?created=1` 이동
- **상태**: `idle` / `loading`(만드는 중…) / `error`(제목 누락·저장 실패)
- **데이터(쓰기)**: `events` INSERT — `slug(랜덤6자), title, theme, starts_at(nullable), place(nullable), description(nullable)`
- **유효성/예외**
  - 제목 공란 → "제목은 필수예요."
  - 미정 토글 ON → 해당 값 `null` 저장
  - DB 실패 → 에러 메시지 노출, 화면 유지
- **비고(H2)**: 날짜·장소 미정 허용이 핵심. 확정 전 초대 가능.

---

### S2. 완성 · 공유 `/e/[slug]?created=1`
- **연결 스토리**: H3, H4
- **목적**: 방금 만든 초대장 확인 + 공유 링크 즉시 제공.
- **구성요소**: 상단 공유 배너("완성! 링크를 카톡방에 붙여요") + `Button` 링크 복사 · 하단은 S3 초대장 미리보기 그대로
- **인터랙션**: [링크 복사] → 클립보드 복사 → "복사됨 ✓" 피드백(1.8s)
- **상태**: `default` / `copied`
- **데이터(읽기)**: `events`(slug로 조회)
- **H4 연결**: 이 링크를 카톡/인스타에 붙이면 **OG 메타(제목·이모지·소개)**가 미리보기 카드로 노출. (커스텀 OG 이미지는 후속 폴리시)
- **비고**: `?created=1` 쿼리로만 배너 표시. 게스트가 여는 순수 링크(S3)에는 안 보임.

---

### S3. 초대장 열람 `/e/[slug]`
- **연결 스토리**: G1
- **목적**: 가입 없이 초대 내용을 감각있게 전달.
- **구성요소**: 테마 히어로(테마색 배경 + 대표 이모지 + 제목 + 한 줄 소개) · 일시/장소 칩(미정이면 "미정") · 하단에 S4 RSVP 카드 연속
- **인터랙션**: 스크롤만으로 열람 → RSVP로 자연 연결
- **상태**: `loaded` / `notFound`(잘못된 slug → 404)
- **데이터(읽기)**: `events`(slug) — 서버에서 조회해 OG 메타 생성
- **예외**: 존재하지 않는 slug → "초대장을 찾을 수 없어요" (404)

---

### S4. RSVP 입력 `/e/[slug]` (열람 하단)
- **연결 스토리**: G2
- **목적**: 최소 마찰로 참석 응답 수집.
- **구성요소**: 안내문("이름만 남겨줘요") · `Input` 이름 · 3버튼 `Button`(🙌 갈게 / 🤔 고민중 / 😢 못가) · 안내("가입·전화번호 없이 · 명단은 서로 볼 수 있어요")
- **인터랙션**
  1. 이름 입력(공란이면 버튼 비활성)
  2. 상태 버튼 탭 → 저장 → S5로 전환
- **상태**: `empty`(이름 없음, 버튼 비활성) / `ready` / `submitting`
- **데이터(쓰기)**: `rsvps` — 최초면 INSERT, 재응답이면 UPDATE (아래 G4)
  - `guest_token`(브라우저 localStorage에 저장한 익명 식별자)로 본인 응답 구분
- **핵심 원칙**: 가입❌ 앱설치❌ 전화번호❌ — **이름 1개 필드만**.

---

### S5. 응답 완료 + 명단 `/e/[slug]` (응답 후)
- **연결 스토리**: G3, G4, H5
- **목적**: 응답 확인 + **실시간 명단으로 소셜 스노볼** 유발 + 호스트 현황 확인.
- **구성요소**
  - 완료 표시("OO님 갈게! 🎉") + [응답 바꾸기]
  - 집계 카운터(갈게 / 고민중 / 못가 수)
  - 상태별 이름 `Badge` 목록 (갈게는 강조)
- **인터랙션**
  - 다른 게스트가 응답하면 **실시간(Realtime)으로 명단 자동 갱신** — 새로고침 불필요
  - [응답 바꾸기] → S4로 복귀 → 기존 응답 UPDATE
- **상태**: `list`(응답자 있음) / `empty`("아직 아무도 없어요")
- **데이터**
  - 읽기: `rsvps`(event_id별) — 상태별 그룹핑·집계
  - 실시간: `rsvps` 테이블 `postgres_changes` 구독(해당 event_id 필터)
- **G4(응답 수정)**: `guest_token` 매칭되는 기존 행을 UPDATE. 재방문(같은 브라우저) 시에도 본인 응답 수정 가능.
- **H5(호스트 현황)**: 별도 대시보드 없이 **이 명단이 곧 호스트의 현황판**. (대시보드는 이번 스코프 제외)

|------|--------|-------|--------|

| S2 완성·공유 | READ | — | — |

### 데이터 모델 (참고)
- `events(id, slug unique, title, theme, starts_at?, place?, description?, created_at)`

## 4. 공통 규칙 & 엣지 케이스
- **모바일 우선**: 모든 화면 `max-w-md` 중앙 정렬, 터치 타깃 넉넉히.
- **무마찰 원칙**: 게스트 경로(S3~S5)에 로그인·가입·전화번호 요구 절대 없음.
- **미정 허용**: 날짜·장소 `null` = "미정" 표기로 일관 처리.
- **익명 식별**: `guest_token`(localStorage)로 재방문 시 본인 응답 수정. 브라우저 바뀌면 새 응답으로 취급(MVP 한계).
- **잘못된 링크**: 존재하지 않는 slug → 404 안내.
- **빈 명단**: 첫 응답 유도 문구 노출.
