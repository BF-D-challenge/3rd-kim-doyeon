# 모디(Modi) — 1주차 미팅 준비

> 목적: 내일 아침 미팅용. PRD의 "1주차: 가정 검증"을 **바로 실행할 수 있는 형태**로 구체화.
> 작성일: 2026-07-11 · 작성자: 김도연
> 연결 문서: [`모디-prd.md`](모디-prd.md)

## ✅ 한눈에 (미팅 3줄 요약)

1. **검증할 것**: "감각 있는 초대 링크를 진짜 쓸까"(Value) + "이름만/명단공개 RSVP에 거부감 없나"(Usability) — 코드 없이 클릭더미로.
2. **MVP 뼈대**: 호스트(템플릿 → 편집 → 공유 → 대시보드) + 게스트(링크 → 열람 → 이름만 RSVP → 명단) 총 8화면.
3. **막히는 것**: 이번주 검증에 쓸 "실제 모임 1건"을 못 정함 → 미팅에서 결정 필요 (후보는 아래 4번에 정리).

---

## 1. 핵심 가정 검증 (Must) — "사람들이 이 초대 방식을 실제로 쓸지?"

### 검증할 Top 2 가정 (PRD에서 가져옴)
| # | 카테고리 | 가정 | 틀리면? |
|---|---------|------|--------|
| A | Value | 호스트가 단톡방/구글폼 대신 "감각 있는 초대 링크"를 진짜 쓰고 싶어 한다 | 한 번 써보고 끝 → 재사용 0, 제품 무의미 |
| B | Usability | 게스트가 가입 없이 **이름만 입력 + 명단 공개**에 거부감 없이 RSVP한다 | 게스트가 이탈 → 집계가 안 모임 |

### 이번주 실험 설계 (코드 없이, 클릭더미로)
- **방법**: 다가오는 실제 모임 1건을 골라, 피그마/노션으로 만든 **가짜 초대 페이지**를 실제 카톡방에 공유.
- **관찰(정량)**:
  - 링크 열람 수 → "참석/미정/불참" 응답 수 (**열람→RSVP 전환율**이 핵심 신호)
  - 응답까지 걸린 시간, 이탈 지점(열고 나감 vs 이름 입력하다 나감)
- **인터뷰(정성) — 게스트 5명**:
  1. "가입/전화번호 없이 이름만 남기는 거 괜찮았어요?"
  2. "참석자 명단이 다른 사람한테도 보이는 거 부담됐어요, 괜찮았어요?"
  3. "이 초대 예뻤어요? 님이 모임 열 때 이런 거 써볼 것 같아요?" (재사용 의향)
- **호스트(본인) 회고**: 단톡방 공지 대비 집계가 실제로 편했는지 1문단.

### 성공/실패 판단선 (사전에 못 박기)
| 신호 | 성공 | 경고 | 실패 |
|------|------|------|------|
| 열람→RSVP 전환 | ≥ 60% | 40~60% | < 40% (마찰 큼) |
| 명단 공개 거부감 (5명 중) | 0~1명 | 2명 | 3명+ (기획 재검토) |
| 재사용 의향 | 3명+ "쓸 듯" | 2명 | 0~1명 |

> 미팅 논의 포인트: 판단선 숫자 동의하는지 / 게스트 5명을 어디서 구할지(4번).

## 2. MVP End-to-End 플로우 (Must) — 템플릿 → 이름만 RSVP → 대시보드

### 호스트 여정 (5화면)
1. **랜딩** — "10초 만에 센스있는 초대장" + [초대장 만들기] CTA
2. **템플릿 갤러리** — 밈 템플릿 6종 썸네일. 클릭 = 온보딩 (설명 페이지 없이 바로 편집)
3. **편집 화면** — 반쯤 채워진 상태로 진입. 편집: 제목(필수) / 날짜·장소(미정 토글) / 한 줄 설명 / 이모지·색 테마
4. **생성 완료 + 공유** — 초대 페이지 미리보기 + [카톡 공유](썸네일 카드) / 링크 복사
5. **호스트 대시보드** — 참석·미정·불참 실시간 카운트 + 게스트 명단 (호스트만 접근)

### 게스트 여정 (3화면)
6. **초대 페이지 열람** — 감성 초대장 렌더링 (날짜/장소/설명 + 분위기)
7. **RSVP 입력** — 이름 한 칸 + [참석 / 미정 / 불참] 3버튼. 가입❌ 앱❌ 전화번호❌
8. **응답 완료 + 명단** — "OO님 참석!" + 현재 참석자 명단 노출 → "쟤 가네? 나도 감" 스노볼

### 화면별 건드리는 데이터 (PRD 스키마 기준)
| 화면 | events | rsvps |
|------|--------|-------|
| 편집(3) → 완료(4) | INSERT (title, template_id, slug…) | — |
| RSVP(7) | READ | INSERT (guest_name, status, guest_token) |
| 완료+명단(8) | READ | READ (event_id별 집계) |
| 대시보드(5) | READ | READ (status별 count + 명단) |

### 결정 필요한 엣지 (미팅 논의)
- 게스트 응답 **수정**은 MVP에 넣나? (PRD: guest_token 로컬 저장 = 경량 지원 방향)
- 링크만 알면 누구나 열람/RSVP → **비공개/인원제한**은 Non-goal로 둘지 확인.

## 3. 초대장 컨셉·템플릿 방향 (Must)

**컨셉 문법: "진지한 척 → 반전 유머"** (타이포·밈 기반, 사진 없이도 성립)

| # | 템플릿명 | 톤 | 대표 카피(초안) | 비주얼 방향 |
|---|---------|----|--------------|-----------|
| 1 | 다들 와야함 | 강압적 유머 | "불참은 개인의 자유. 근데 서운할 예정." | 큰 타이포, 굵은 대비색 |
| 2 | 불참시 다음생에 | 협박(개그) | "이번 생에 못 보면 다음 생에 봅니다." | 어두운 배경 + 네온 |
| 3 | 조용히 스며들기 | 소심·귀여움 | "부담 갖지 마세요. 그냥… 오세요." | 파스텔, 여백 많음 |
| 4 | 생일인 척 | 능청 | "제 생일은 아닌데, 축하받고 싶어서요." | 컨페티, 발랄 |
| 5 | 집들이(핑계) | 현실 개그 | "집 자랑 아니고요, 사람 그리워서요." | 따뜻한 우드톤 |
| 6 | 일단 모여 | 무계획 | "계획 없음. 장소만 있음. 일단 와." | 미니멀 흑백 |

> 미팅 논의: 6종 다 만들지 vs **1주차 검증은 1~2종만** (PRD 2주차와 정렬). / 카피 톤이 타겟(20-30대 IT직장인)에 맞는지 감수.

## 4. 🚧 막히는 것 — 이번주 검증 대상(모임/사람) 확보

**문제**: 검증하려면 "실제로 다가오는 모임 1건 + 게스트 5명"이 필요한데 아직 미정.

**후보 옵션 (미팅에서 하나 고르기)**
- **A. 본인 주최 실모임** — 집들이/생일/저녁 번개 등 이번주~다음주 안에 실제로 열 것. (가장 진짜 신호, but 일정 필요)
- **B. 챌린지 동료 모임** — BF.D 스터디/회식에 클릭더미 초대 붙여 실험. (섭외 쉬움, 게스트가 "제품 만드는 사람들"이라 신호 왜곡 주의)
- **C. 지인 모임에 얹기** — 친구가 여는 모임 호스트에게 "초대 이 링크로 해볼래?" 부탁. (진짜 호스트 관점까지 관찰 가능)
- **D. 가상 모임 + 인터뷰만** — 실제 모임 없이 클릭더미 보여주고 5명 반응만. (빠름, but 열람→RSVP 실전환 신호 없음 → 최후수단)

> 추천: **A 또는 C** (실제 RSVP 전환을 봐야 Value·Usability 둘 다 검증됨). B는 보조.
> 결정하면: 대상 모임 날짜 / 게스트 풀 / 언제 클릭더미 공유할지 타임라인 픽스.

## 5. 💡 레퍼런스/벤치마크 (Nice) — RSVP 흐름·공유 카드

| 서비스 | 볼 점 | 모디에 시사점 |
|--------|------|--------------|
| **Partiful** | 가입 최소·문자 RSVP·"누가 오는지" 명단 강조 | 명단 공개 스노볼의 원조격 — 우리 8번 화면 벤치 |
| **Luma (lu.ma)** | 깔끔한 이벤트 페이지 + 캘린더 연동 | 편집/렌더 화면 UX 참고 (단, 우린 더 밈스럽게) |
| **Paperless Post** | 감성 초대장 "카드" 비주얼 | 템플릿 무드보드 레퍼런스 |
| **인스타 스토리 초대 스티커** | 탭 한 번 참여, 무마찰 | RSVP 마찰 최소화 벤치 |
| **구글폼/네이버폼** | (반면교사) 여러 항목·무드 없음 | 우리가 안 할 것의 기준선 |
| **카카오 톡캘린더/오픈채팅 공지** | 진짜 경쟁자(단톡방)의 집계 한계 | "왜 단톡방으론 안 되나" 근거 |

> 리서치 액션: 위 6개 중 **Partiful·Luma 2개만** RSVP 화면 캡처해 무드보드 만들기 (30분).
