## 0. Partiful 플로우 요약 (관찰)
```
스플래시 → 홈 피드(Upcoming/Hosting 탭) → [+] 생성
생성: 커버이미지(검색/GIF/업로드) → 제목·날짜(미정→게스트 투표) → 호스트닉네임/공동호스트
      → 장소 · 정원(+대기자) · 1인 비용(Chip In) · 게스트 초대허용 · RSVP마감
      → +Link/Playlist/Registry/DressCode · 설명 · [Theme/Effect/Settings]
공유: Compose(문구 템플릿 "Hey [Name]…RSVP at [link]") · Email 초대(대량)
게스트: 초대 페이지 → RSVP(Going👍/Maybe🤔/Can't Go😔) → Questionnaire → 명단
관리: Manage Guests(검색·상태필터·정렬) · Photo Album · Clone/Make Flyer/Cancel
소셜: 프로필·팔로워·Boop·채팅("You both RSVP'd to…")
```

## 1. 핵심 관찰 포인트
| 패턴 | Partiful 방식 | 우리에게 주는 힌트 |
|------|--------------|-------------------|
| **커버 비주얼** | 이미지 검색(Trending/Birthday/Minimal)+GIF+업로드 | "감각/재미"의 최대 지렛대. 우리 테마색+이모지보다 한 단계 위 |
| **Effect(움직임)** | 테마 위에 버블 등 애니메이션 오버레이 | 저비용 고감성 — 기대감 부스터 |
| **날짜 미정** | "Can't decide? Poll your guests" | 미정을 그냥 두지 않고 게스트 투표로 전환 |
| **RSVP 버튼** | Going/Maybe/Can't Go 이모지, 스타일 커스텀 | 우리 갈게/고민중/못가와 동일 ✓ |
| **정산(Chip In)** | 금액+Venmo/CashApp/PayPal 핸들. **"결제 미검증, 게스트 자가신고"** | 돈 안 만지는 self-report 구조 = 규제 리스크 없음 (아래 3) |
| **동반/정원** | Plus Ones, Max Capacity+Waitlist, 호스트 승인 | Phase 2 확장 후보 |
| **Questionnaire** | RSVP 때 커스텀 질문(드롭다운/필수) | 식단·이메일 등 수집 |
| **공유 문구** | 자동 메시지 템플릿 | 호스트 마찰↓ |
| **사진 앨범** | 모임 후 공동 앨범 | 모임 후 재방문 = 재사용 지표↑ |
| **소셜(계정·팔로워·채팅)** | 앱 네트워크 | ❌ 우리 무가입 원칙과 충돌 |

## 1.5 Apple Invites 벤치마크 (2026-07 추가 — 직접 사용 후)

> 우리와 **구조가 가장 비슷한** 레퍼런스. 2025년 초 출시, iCloud+ 기능.

| 패턴 | Apple Invites 방식 | 우리에게 주는 힌트 |
|------|-------------------|-------------------|
| **비대칭 구조** | 생성=iCloud+ 구독자만 / **RSVP=계정·기기 무관 웹에서 누구나** | ✅ 우리 "호스트 계정 + 게스트 무가입" 모델을 애플이 검증해줌 |
| **RSVP + note** | Going/Not going/Maybe + **선택적 한마디** — 호스트·게스트 모두에게 노출 | ✅ "한마디" 필드 확신. 명단→대화 전환 |
| **호스트 공지** | 호스트가 이벤트에 메시지 게시 → 전원에게 보임 | H4 확정 알림과 결합할 만한 패턴 (변경사항 공지) |
| 게스트 프라이버시 | 내 정보 노출 범위 제어 + 이벤트 나가기/신고 | P2 참고 (명단 공개 거부감 대응) |
| 공동 앨범·플레이리스트 | RSVP 후 참여 가능 | 기존 백로그 항목과 동일 (모임 후 재방문) |
| 커버 | 사진/이모지 풀블리드 + 타이포 | 우리 사진 커버와 같은 결 |
| 한계 | 확정 일정 중심(미정→확정 루프 없음) · 한국 카톡 문화 미지원 · 호스트 유료 장벽 | **우리 차별점**: 미정 게시→확정 알림 루프 + 카톡 OG + 무료 |

## 2. 지금(Adopt now) — MVP에 얹을 저비용 고감성
> 무가입·무마찰 원칙 유지 + 스키마 거의 안 건드리는 것 위주.
- **A. Effect/비주얼 flair** — 테마별 움직이는 요소(떠다니는 이모지/은은한 그라데이션). 순수 프론트, "오 뭐야" 반응 강화.
- **B. 공유 문구 템플릿** — 완성 화면에서 카톡 복붙용 메시지 자동 제공 (예: "OO가 초대했어요 · [제목] 👉 [링크]"). 순수 프론트.
- **C. (선택) 'Hosted by' 호스트 이름** — 소소한 온기. `events.host_name` 컬럼 1개 추가.

## 3. Phase 2 — Partiful이 검증해준 확장 (정산 포함)
### 3-1. 정산·예약금 (Chip In의 한국판) ← 사용자 요청
- **원칙(Partiful 동일)**: 우리가 돈을 만지지 않는다. **"결제 미검증 · 게스트 자가신고"**. → 전자금융 라이선스 불필요, 리스크 낮음.
- **호스트(생성 시, 옵션 섹션)**: 금액 + 정산수단 선택
  - 토스/카카오페이 **송금 딥링크**(금액·계좌 자동 채움) 또는 **계좌(은행·번호·예금주) + 복사**
- **게스트(초대장)**: "1인 12,000원 · [토스로 보내기] / [계좌 복사]" 노출
- **예약금(노쇼 방지)**: RSVP 시 동일 UI로 "참석=예약금 송금" 안내(자가신고)
- **데이터(안)**: `events`에 `cost_amount, cost_currency, pay_method('bank'|'toss'|'kakaopay'), pay_handle`; (후속) `rsvps.paid boolean` 호스트 수동 체크
- **부수효과**: 모임 후에도 초대장 재방문 → 재사용(북극성 Input)에 긍정적

### 3-2. 기타 확장 후보
- 동반인원(Plus Ones) · 정원+대기자(Capacity/Waitlist) · 호스트 승인 · RSVP 마감
- Questionnaire(커스텀 질문) · 공동 사진 앨범 · 커버 이미지 업로드(Supabase Storage) · 날짜 투표

## 4. 제외(Not for us)
- 계정·프로필·팔로워·Boop·채팅·mutuals·홈 피드 — **무가입 원칙과 충돌.** 우리는 "링크 하나, 가입 없음"을 지킨다.
