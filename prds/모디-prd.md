## 1. 문제 정의

### 표면 문제
직장인들이 시간 내서 여는 사적 모임(친구 생일·술약속·놀러가기)의 초대가 **밋밋한 단톡방 공지**에 머물러서, 모임 전 기대감·재미가 죽는다. (실제 경험: 친구 생일인데 단톡방 공지가 너무 밋밋해서 김샜음)

### 본질적 문제 (5 Whys로 도출)
사적 모임에 **테마·주제를 입혀 "초대 자체가 설레고, SNS에 공유하고 싶은 콘텐츠"가 되게 해주는 도구가 없다.** 한국에서 뭐든 "파티"라 부르며 모이는 문화(강남 IT파티·솔로파티…)가 커지는데, 그 "모이는 문화"를 더 센스있게 호스팅하도록 거드는 앱이 비어 있다.

### 현재 사람들이 푸는 방식 (As-is)
- 밋밋한 단톡방 공지 (테마·기대감 없음)
- 구글폼/네이버폼 (사무적, 무드 없음)
- 인스타 스토리 초대 스티커 (가볍지만 테마·집계 약함)
- 노션 페이지 공유 (예쁘지만 모바일·집계 약함)

### Why now?
- 한국 파티·소셜 모임 문화 확산 (강남 IT파티, 솔로파티 등 "파티" 워딩 일상화)
- SNS로 "감각 있는 초대장 공유"가 자연스러워짐
- 링크 OG 카드로 "감성 초대 미리보기"를 API 연동 없이도 붙일 수 있게 됨

## 2. 타겟 & JTBD

### 페르소나
**수아 / 28세 / IT업계 직장인(마케터·디자이너 등).** 두 달에 한두 번 친구 생일·집들이·주말 번개를 주최한다. 단톡방에 "토요일 7시 우리집~" 올리는데 너무 밋밋해서 스스로도 김샌다. 인스타 스토리에 감각 있는 초대 카드 올리는 친구들 보면서 "나도 저렇게 센스있게 부르고 싶다" 하지만, 디자인 툴 붙잡고 만들 시간·여력은 없다.

### Job Story
When **친구들 불러 사적 모임을 열 때**,
I want to **모임에 테마를 입힌 초대장을 10초 만에 만들어 링크로 보내**,
so I can **친구들이 "오 뭐야 ㅋㅋ 가고싶다" 하며 모임 전부터 들뜨게 만든다.**

### Value Proposition
- **Who**: 친구·지인 사적 모임을 자주 여는 20-30대 IT업계 직장인 호스트
- **Why**: 초대 자체를 재밌고 감각있게 해서 기대감을 만들고, SNS에 자랑도 하고 싶다
- **What before**: 밋밋한 단톡방 공지 → 사무적, 기대감 0
- **How**: 테마 템플릿으로 10초 만에 초대장 생성 → 링크/OG카드로 공유 → 게스트가 설레고 웃음
- **What after**: 보내는 순간 "센스있다" 소리 듣고, 친구들이 모임 전부터 들뜨고, SNS 공유각까지
- **Alternatives**: 단톡방(밋밋), 인스타 스토리 스티커(테마·집계 약함), 구글폼(사무적), 직접 디자인(시간 없음)

## 3. 임팩트 가설

### North Star Metric
**살아있는 초대 수** — 만들어서 실제로 공유됐고, 게스트 반응(RSVP 또는 리액션)이 **3개 이상** 달린 초대장 수. (단순 생성 수는 허수로 제외 — 실제로 기대감을 일으킨 초대만 셈)

### Input Metrics
1. **🔥 SNS 외부 재공유율** — "자랑하고 싶었다"의 신호이자 확산 엔진 (재미의 결과)
2. **생성 → 공유 전환율** — 만들고 실제로 남한테 보냈나
3. **열람 → 반응 전환율** — 게스트가 열고 설레서 반응했나 (재미의 핵심 신호)

### 정성적 임팩트
> "이걸로 초대하니까 애들이 '뭐야 이거 ㅋㅋ' 하면서 바로 온다더라."

## 4. 솔루션 (MVP)

### 핵심 가치 한 문장
IT업계 20-30대 직장인 호스트를 위한 **테마 감성 초대장 웹앱**인데, Partiful처럼 놀 거 같은 초대를 10초 만에 만들되 **게스트가 가입·전화번호 없이 이름만으로 RSVP**해서 마찰이 가장 낮다.

### Must-have 기능
1. **테마 템플릿으로 초대장 생성** — 제목만 필수(날짜·장소 미정 허용), 테마 선택 → 초대 페이지 자동 생성
2. **무마찰 게스트 RSVP + 명단** — 링크 클릭 → 이름만 입력 → 참석/미정/불참, 참석자 명단 노출("쟤 오네? 나도" 스노볼). 호스트도 이 명단으로 현황 파악
3. **링크 + OG 카드 공유** — 카톡/인스타/슬랙에 링크 붙이면 자동 미리보기 카드

### Non-goals (이번엔 안 할 것)
- **호스트 대시보드** — 게스트용 명단으로 대체 (집계는 이번 방향에서 우선순위 낮음)
- **카카오 SDK 네이티브 연동** — OG 카드로 대체
- **파티(B) 기능** — 오픈 모집·낯선 사람 신청·인원 필터는 확장 단계로 (A: 친구 사적모임 먼저)
- 로그인·회원가입 / 결제·정산 / 네이티브 앱 / 푸시 알림

### Demo Flow (4주차 시연)
1. 템플릿 갤러리에서 테마 클릭 → 반쯤 채워진 편집 화면
2. 제목·날짜 입력 → 초대 페이지 생성 (10초)
3. 링크 복사 → 카톡방에 붙임 → OG 미리보기 카드 뜸
4. 게스트 폰: 링크 클릭 → 이름만 입력 → "참석" → 명단에 실시간 반영
5. 다른 게스트가 명단 보고 "나도 감" → 반응 쌓임

## 5. 리스크 & 가정

### VUVF 가정 매핑

| 카테고리 | 가정 |
|----------|------|
| Value | 사람들이 밋밋한 단톡방 대신 "테마 초대장"을 진짜 쓰고 싶어 하고, 한 번 재미로 끝이 아니라 또 쓴다 |
| Usability | 게스트가 가입·전화번호 없이 이름만 + 명단 공개에 거부감 없이 RSVP한다 |
| Viability | 주말 8h × 4주로 생성 + RSVP + OG공유 3개를 만들 수 있다 |
| Feasibility | 링크 붙이면 카톡에서 OG 카드가 안정적으로 뜬다 (스크래핑·이미지 캐싱 이슈 없이) |

### 가장 위험한 가정 (Top 1~2)
1. **(Value)** "테마의 재미"가 재사용을 만들 만큼 매력적인가 — 일회성 장난감이면 죽음
2. **(Usability)** 게스트가 명단 공개 + 이름 입력에 거부감 없이 응답하나 — 막히면 반응(북극성)이 안 쌓임

### 1주차 검증 실험 (코드 없이 / 라이브 테스트)
플로우 설계 → 가벼운 프로토타입(피그마 클릭더미 또는 최소 웹) → **내일 오후 실제 약속에서 직접 써봄** → 게스트 반응·마찰 지점 현장 관찰 (열람→반응 전환 / "재밌다" 리액션 / 이름·명단 거부감).

## 6. 주차별 마일스톤

### 1주차: 가정 검증 (라이브 테스트)
- **목표**: 플로우 설계 + 가벼운 프로토타입 → 내일 오후 약속에서 라이브 테스트
- **산출물**: 프로토타입 + 현장 관찰 노트 (반응·마찰 지점)

### 2주차: 핵심 기능 구현
- **목표**: 테마 템플릿으로 초대장 생성 (실제 웹 구현 시작)
- **산출물**: 배포된 링크로 초대장이 생성되는 화면

### 3주차: 통합 & 추가 기능
- **목표**: 이름만 RSVP + 명단 + 링크/OG 카드 공유 통합
- **산출물**: 실폰에서 생성 → RSVP → 명단 → 공유 전체가 동작하는 상태

### 4주차: 데모 & 발표
- **목표**: 데모 시나리오 완성 + 실제 모임으로 리허설
- **산출물**: 데모 영상 + 발표 자료
