## 한 줄 정의

**초대장을 "만드는 재미"가 곧 기대감이 되는 앱.** 밋밋한 단톡방 공지 대신, 모임에 테마를 입혀 꾸민 초대장 하나로 — 보내는 사람은 "나 센스있다", 받는 사람은 "오 뭐야 ㅋㅋ".

> 📌 **핵심 피벗(2026-07-12)**: 소수·친밀한 모임에선 *참석 집계(RSVP)가 거의 무의미*하다 (친구가 못 오면 사정이 있는 거지 관리할 일이 아님). 진짜 가치는 **"초대장을 감각있게 꾸미는 표현 경험"** 에 있다. → 제품의 심장을 **꾸미기**로, RSVP는 **가벼운 이모지 반응**으로 강등.

## 1. 문제 정의

### 표면 문제
직장인들이 시간 내서 여는 사적 모임(친구 생일·술약속·놀러가기)의 초대가 **밋밋한 단톡방 공지**에 머물러서, 모임 전 기대감·재미가 죽는다. (실제 경험: 친구 생일인데 단톡방 공지가 너무 밋밋해서 김샜음)

### 본질적 문제 (5 Whys로 도출)
사적 모임에 **테마·주제를 입혀 "초대 자체가 설레고, SNS에 공유하고 싶은 콘텐츠"가 되게 해주는 도구가 없다.** 특히 소수·친밀한 모임에선 참석 여부를 세는 게 아니라, **"이 자리를 특별하게 마킹하고 표현하는 것"** 이 진짜 니즈다. 뭐든 "파티"라 부르며 모이는 문화(강남 IT파티·솔로파티…)가 커지는데, 그 문화를 더 센스있게 호스팅하도록 거드는 앱이 비어 있다.

### 진짜 경계선 (인원 수가 아니라 "자리의 특별함")
| 상황 | 초대장 꾸미기 | RSVP 집계 |
|------|:---:|:---:|
| 3명 그냥 저녁 | ❌ (그냥 카톡) | ❌ |
| 3명 깜짝 생일 | ✅ **핵심** | ❌ 무의미 |
| 15명 집들이·파티 | ✅ 핵심 | 🔶 가벼운 반응 정도 |

→ 가치의 축은 **"캐주얼 만남 vs 마킹할 만한 이벤트"**. 우리는 후자를 겨냥한다.

### 현재 사람들이 푸는 방식 (As-is)
- 밋밋한 단톡방 공지 (테마·기대감 없음)
- 인스타 스토리 초대 스티커 (가볍지만 꾸미기·테마 약함)
- 노션/구글폼 (사무적, 무드 없음)
- 직접 디자인(피그마/캔바) — 예쁘지만 시간·여력 필요

### Why now?
- 한국 파티·소셜 모임 문화 확산 + SNS로 "감각 있는 초대장 공유"가 자연스러워짐
- 링크 OG 카드로 "감성 초대 미리보기"를 API 연동 없이 붙일 수 있게 됨

## 2. 타겟 & JTBD

### 페르소나
**수아 / 28세 / IT업계 직장인(마케터·디자이너 등).** 두 달에 한두 번 친구 생일·집들이·주말 번개를 주최한다. 단톡방에 "토요일 7시 우리집~" 올리는데 너무 밋밋해서 스스로도 김샌다. 인스타 스토리에 감각 있는 초대 카드 올리는 친구들 보면서 "나도 저렇게 센스있게 부르고 싶다" 하지만, 디자인 툴 붙잡고 만들 시간·여력은 없다.

### Job Story
When **친구들과의 자리를 좀 특별하게 열고 싶을 때**,
I want to **모임에 테마를 입힌 초대장을 10초 만에 꾸며 링크로 보내**,
so I can **보내는 순간 센스있어 보이고, 받는 친구들이 "오 뭐야 ㅋㅋ" 하며 모임 전부터 들뜨게 만든다.**

### Value Proposition
- **Who**: 친구·지인과의 "마킹할 만한 자리"를 자주 여는 20-30대 직장인 호스트
- **Why**: 초대 자체를 감각있게 **꾸며서** 기대감을 만들고, SNS에 자랑도 하고 싶다
- **What before**: 밋밋한 단톡방 공지, 또는 디자인 툴 붙잡고 낑낑 → 사무적이거나 시간 소모
- **How**: 테마 + 커버·색·문구·스티커·이펙트로 **10초 만에 꾸민 초대장** → 링크/OG카드로 공유
- **What after**: 보내는 순간 "센스있다", 친구들이 들뜨고, SNS 공유각까지
- **Alternatives**: 단톡방(밋밋), 인스타 스티커(꾸미기 약함), 캔바/피그마(무거움), 구글폼(사무적)

## 3. 임팩트 가설

### North Star Metric
**설레는 초대 수** — 만들어서 실제로 **공유됐고**, 게스트 **이모지 반응이 3개 이상** 달렸거나 **외부(SNS)로 재공유된** 초대장 수. (단순 생성 수 제외 — 실제로 반응/확산을 만든 초대만 셈)

### Input Metrics
1. **🔥 SNS 외부 재공유율** — "자랑하고 싶었다"의 신호이자 확산 엔진
2. **생성 → 공유 전환율** — 꾸미고 실제로 남한테 보냈나
3. **열람 → 반응 전환율** — 게스트가 열고 이모지 반응했나 (재미의 핵심 신호)
4. **재사용 호스트 수** — 한 번 쓰고 또 만드는가 (표현 경험이 재미있었나)

### 정성적 임팩트
> "이걸로 초대하니까 애들이 '뭐야 이거 ㅋㅋ 예쁘다' 하면서 캡처해서 자기 스토리에 올리더라."

## 4. 솔루션 (MVP) — 꾸미기가 핵심

### 핵심 가치 한 문장
20-30대 직장인 호스트를 위한 **초대장 꾸미기 웹앱**인데, 디자인 툴 없이 **테마+커버+문구+스티커로 10초 만에 감각있는 초대장**을 만들어 링크로 공유하고, 게스트는 가입 없이 **이모지로 반응**한다.

### Must-have 기능
1. **🎨 초대장 꾸미기 (핵심)** — 테마 선택 + 커버(색/그라데이션/이미지) + 제목·문구 + 이모지 스티커 + 움직이는 이펙트. 제목만 필수, 날짜·장소 미정 허용. **이 경험이 제품의 심장.**
2. **링크 + OG 카드 공유** — 카톡/인스타에 링크 붙이면 자동 미리보기 카드 + 복붙용 공유 문구
3. **가벼운 이모지 반응 (RSVP 강등)** — 게스트가 가입·이름 강제 없이 **하트·이모지로 반응**, (선택) "갈게" 태그. **집계·관리 목적 아님 — 재미와 소셜 증거(누가 반응했나)용.**

### Non-goals (이번엔 안 할 것)
- **RSVP 관리·명단 집계·호스트 대시보드** — 참석 관리는 이 방향의 핵심이 아님 (가벼운 반응으로 대체)
- **모임 라이프사이클(날짜 조율·캘린더·정산)** — 리텐션 확장 축이지만 **Phase 2** (아래 로드맵)
- 카카오 SDK 네이티브 연동(OG로 대체) / 로그인·회원가입 / 반(半)공개 파티 모집 / 네이티브 앱

### Demo Flow (4주차 시연)
1. 테마 선택 → 커버·문구·스티커·이펙트로 **꾸미기** (10초 컷)
2. 완성 → 링크 + 공유 문구 복사 → 카톡방에 붙임 (OG 카드 뜸)
3. 게스트 폰: 링크 클릭 → 꾸며진 초대장 감상 → **이모지 반응** 탭
4. 반응이 초대장에 실시간으로 쌓이고, 게스트가 캡처/재공유

## 5. 리스크 & 가정

### VUVF 가정 매핑
| 카테고리 | 가정 |
|----------|------|
| Value | 사람들이 "초대장을 꾸미는 재미"에 끌리고, 그게 **공유·재사용**을 만든다 |
| Usability | 디자인 지식 없이도 10초 만에 "센스있어 보이는" 초대장이 나온다 (꾸미기가 쉽고 결과가 예쁨) |
| Viability | 주말 8h × 4주로 꾸미기 코어 + 공유 + 반응을 만들 수 있다 |
| Feasibility | 링크 붙이면 카톡에서 OG 카드가 안정적으로 뜬다 |

### 가장 위험한 가정 (Top 1~2)
1. **(Value)** "꾸미는 재미"가 재사용·SNS 공유를 만들 만큼 매력적인가 — 한 번 만들고 끝이면 노벨티로 사망
2. **(Usability)** 아무나 10초 만에 **"오 예쁘다" 소리 나오는** 결과를 뽑을 수 있나 — 못생기게 나오면 안 씀

### 1주차 검증 실험 (라이브 테스트)
프로토타입을 **내일 오후 실제 약속에서 써봄** → 관찰: 초대장 보고 "예쁘다/뭐야ㅋㅋ" 반응이 나오나, 캡처·공유하나, (RSVP보다) 꾸밈에 반응하나.

## 6. 주차별 마일스톤

### 1주차: 가정 검증 (라이브 테스트)
- **목표**: 프로토타입 → 내일 오후 약속에서 라이브 테스트
- **산출물**: 프로토타입 + 현장 관찰 노트 (꾸밈 반응·공유 여부)

### 2주차: 꾸미기 코어 구현
- **목표**: 테마 + 커버(색/이미지) + 문구 편집 + 이모지 스티커 = "꾸미는 경험" 완성
- **산출물**: 배포 링크로 초대장을 실제로 꾸며 만드는 화면

### 3주차: 이펙트 + 공유 + 반응
- **목표**: 움직이는 이펙트 + 링크/OG 공유 문구 + 게스트 이모지 반응 통합
- **산출물**: 실폰에서 꾸미기 → 공유 → 반응 전체 동작

### 4주차: 데모 & 발표
- **목표**: 데모 시나리오 완성 + 실제 모임으로 리허설
- **산출물**: 데모 영상 + 발표 자료
