## 한 줄 정의

**초대장이 곧 기대감이 되는 앱.** 카톡 단톡방 공지 + 구글폼으로 흩어진 모임 초대를, 보내는 순간 "나 센스있다" 싶은 초대 페이지 하나로.

---

## 1. 문제 정의

### 표면 문제

모임을 여는 호스트가 초대·집계·공지를 **카톡 단톡방**과 **구글폼**에 나눠서 처리하느라, 참석 현황이 채팅에 파묻히고 명단 관리가 번거롭다.

### 본질적 문제

진짜 경쟁자는 다른 초대 앱이 아니라 **카톡 단톡방**이다. 단톡방은 다음 세 가지를 동시에 잘 못한다:

1. **집계** — "저요", "저도요", "저는 미정..." 이 스크롤에 묻혀 호스트가 직접 센다
2. **공지** — 날짜/장소가 바뀌면 다시 올려야 하고, 늦게 들어온 사람은 위로 스크롤해야 함
3. **기대감** — 텍스트 공지엔 "가고 싶다"는 감정이 안 실린다. 초대가 사무적이다

즉, 문제의 본질은 *"집계·공지·기대감을 한 곳에서, 게스트 마찰 없이"* 처리하는 도구의 부재다.

### 현재 사람들이 푸는 방식 (As-is)

- 카톡 단톡방 공지 + 손으로 참석자 세기
- 구글폼/네이버폼으로 RSVP 받기 (게스트는 폼 열고 여러 항목 입력해야 함 → 이탈)
- 노션 페이지 공유 (예쁘지만 모바일 열람·집계가 약함)

### Why now?

- 20-30대 IT/스타트업 직장인 사이에서 **소모임·번개·집들이** 빈도가 높아지고, "무드 있는 초대"에 대한 기대치가 올라감 (인스타 스토리 초대 카드 문화)
- 카카오톡 공유 SDK로 **썸네일 카드 미리보기**를 손쉽게 붙일 수 있어, 링크 하나로 "감각 있는 초대"가 기술적으로 가능해짐

## 2. 타겟 & JTBD

### 타겟

20-30대 IT/스타트업 직장인 호스트 (생일, 집들이, 팀 회식, 소모임 주최자).
공유 채널 우선순위: **카카오톡 > 인스타 DM > 슬랙**

### 페르소나

**지민 / 28세 / 스타트업 프로덕트 디자이너.** 두 달에 한 번쯤 집들이·생일·소모임을 주최한다. 단톡방에 공지 올리고 "참석 여부 이모지로 남겨주세요" 하지만, 늘 몇 명 올지 몰라 음식·자리 예약을 애매하게 한다. 초대할 때 "좀 더 센스 있어 보이고 싶은데" 하는 아쉬움이 있다.

### Job Story

> When **소모임을 열어 사람들을 초대할 때**,
> I want to **링크 하나로 감각 있게 초대하고 참석 현황을 자동으로 집계**,
> so I can **몇 명 올지 확신하고, 초대 자체로 기대감을 만들 수 있다**.

### Value Proposition

- **Who**: 소규모 모임을 자주 여는 20-30대 IT/스타트업 직장인 호스트
- **Why**: 초대·집계·공지를 한 번에, 게스트 마찰 없이 처리하고 싶다
- **What before**: 단톡방 공지 + 수기 집계 + 구글폼 → 흩어지고 사무적
- **How**: 밈 템플릿으로 10초 만에 초대 페이지 생성 → 카톡 썸네일 카드로 공유 → 게스트는 이름만 입력해 RSVP → 호스트 대시보드에서 실시간 집계
- **What after**: 초대를 "보내는 순간 센스 있어 보이고", 참석 인원을 확신하며, 게스트도 명단을 보고 "나도 감" 하게 됨
- **Alternatives**: 카톡 단톡방(집계·기대감 약함), 구글폼(마찰 큼·무드 없음), 노션(모바일 집계 약함)

## 3. 임팩트 가설

### North Star Metric

**살아있는 이벤트 수** — RSVP가 **3명 이상** 모인 이벤트 수. (단순 생성 수는 허수로 보고 집계에서 제외)

### Input Metrics

1. **초대 생성 → 최초 공유 전환율** (만들고 실제로 공유까지 간 비율)
2. **링크 열람 → RSVP 전환율** (게스트 마찰이 낮은지 보는 핵심 신호)
3. **재방문 호스트 수** (한 번 쓰고 다음 모임에도 다시 쓰는지)

### 정성적 임팩트

> "이걸로 초대하니까 사람들이 '오 뭐야 이거' 하면서 바로 참석 누르더라."

## 4. 솔루션 (MVP)

### 핵심 가치 한 문장

*소모임을 자주 여는 20-30대 호스트를 위한 **밈 감성 초대장 웹앱**인데, 게스트가 가입·앱설치·전화번호 없이 이름만으로 RSVP할 수 있어 마찰이 가장 낮다.*

### Must-have 기능

#### 1) 이벤트 생성 + 밈 템플릿 6종

- **제목만 필수** (날짜·장소는 "미정" 허용) → 초대 페이지 자동 생성
- 타이포/밈 기반 템플릿 6종. "진지한 척 → 반전 유머" 문법.
  - 예: "다들 와야함", "불참시 다음생에 만남", "조용히 스며들기", "생일인 척", "집들이(핑계)", "일단 모여"
- 템플릿 갤러리에서 클릭 → **반쯤 완성된 초대장 편집 화면**으로 바로 진입 (템플릿 = 온보딩)
- 편집 항목: 제목, 날짜/시간(미정 토글), 장소(미정 토글), 한 줄 설명, 대표 이모지/색상 테마

**데이터(초안)**: `events(id, host_id, template_id, title, starts_at nullable, place nullable, description, theme, slug unique, created_at)`

#### 2) 무마찰 게스트 RSVP

- 링크 클릭만으로 **열람 + RSVP**. 가입 ❌, 앱 다운 ❌, **전화번호도 ❌**
- 이름만 입력하면 끝. 상태 3종: **참석 / 미정 / 불참**
- RSVP 후 **참석자 명단이 게스트에게도 노출** → "쟤 가네? 나도 감" 소셜 스노볼
- (경량) 브라우저에 게스트 로컬 식별자를 저장해 재방문 시 본인 응답 수정 가능

**데이터(초안)**: `rsvps(id, event_id, guest_name, status enum('going','maybe','no'), guest_token, created_at, updated_at)`

#### 3) 원탭 공유 + 호스트 대시보드

- **카카오톡 공유 SDK로 썸네일 카드 공유** (미리보기 이미지 필수) · 인스타 · 슬랙 공유
- 호스트 대시보드: **참석/미정/불참 실시간 집계 + 게스트 명단** 확인
- 링크 하나로 열람/공유/RSVP가 모두 동작 (호스트만 대시보드 접근)

### Non-goals (이번엔 안 할 것)

- **더치페이 정산** → Phase 2 (결제 연동 리스크, MVP 검증엔 불필요)
- **알림톡 리마인더** → 비즈채널 심사 지연 리스크, 호스트 수동 공지로 대체
- **iOS 네이티브 앱** → 웹 반응 보고 Expo 전환 판단
- **공동 사진 앨범 · 날짜 투표 · 이모지 반응** → 코어 검증 후
- **수익화** → MVP는 무조건 무료
- **정식 브랜드 네이밍** → 코드네임 "모디"로 개발 진행

### Demo Flow (4주차 시연)

1. 템플릿 갤러리에서 "다들 와야함" 클릭 → 반쯤 완성된 초대장에서 시작
2. 제목·날짜·장소 입력 → 초대 페이지 생성 (10초 컷)
3. 카톡 공유 버튼 → 실제 폰 카톡방에 썸네일 카드 전송
4. 게스트 폰: 링크 클릭 → 가입 없이 이름만 입력 → "참석" → 명단에 실시간 반영
5. 호스트 대시보드에서 참석/미정/불참 집계 확인

## 5. 리스크 & 가정

### VUVF 가정 매핑

| 카테고리 | 가정 |
|----------|------|
| Value | 호스트가 단톡방/구글폼 대신 "감각 있는 초대 링크"를 진짜 쓰고 싶어 한다 |
| Usability | 게스트가 가입 없이 이름만으로 RSVP하는 흐름이 실제로 마찰 없이 돈다 |
| Viability | 4주 안에 생성 + RSVP + 카톡 공유 + 대시보드까지 만들 수 있다 |
| Feasibility | 카카오 공유 SDK 썸네일 카드 미리보기를 안정적으로 붙일 수 있다 |

### 가장 위험한 가정 (Top 1~2)

1. **(Value)** "감각 있는 초대"가 재사용을 만들 만큼 매력적인가 — 한 번 써보고 끝나면 실패
2. **(Usability)** 게스트가 명단 공개 + 이름 입력에 거부감 없이 RSVP하는가

### 1주차 검증 실험 (코드 없이)

- 실제 다가오는 모임 1건을 골라, **손으로 만든 초대 페이지(피그마/노션 클릭더미)** 를 카톡방에 공유
- 게스트가 "참석" 의사를 어디에 어떻게 남기는지 관찰 + 5명에게 "가입 없이 이름만 남기는 거 괜찮아요?" 인터뷰
- 지표: 링크 열람 → 참석 응답 전환, "이거 예쁘다/또 쓰고 싶다" 반응 수집

## 6. 주차별 마일스톤

### 1주차: 가정 검증

- **목표**: 손으로 만든 초대 페이지로 실제 모임 1건 초대 실험 + 게스트 5명 인터뷰
- **산출물**: 검증 결과 정리 문서 (열람→RSVP 전환, 명단 공개 거부감 여부)

### 2주차: 핵심 기능 구현

- **목표**: 이벤트 생성 + 초대 페이지 렌더링 + 이름만 입력 RSVP (템플릿 1~2종)
- **산출물**: 배포된 링크로 생성→RSVP가 동작하는 화면

### 3주차: 통합 & 추가 기능

- **목표**: 템플릿 6종 갤러리 + 카카오 공유 썸네일 카드 + 호스트 대시보드 실시간 집계
- **산출물**: 데모 플로우 전체가 실제 폰에서 도는 상태

### 4주차: 데모 & 발표

- **목표**: 데모 시나리오 완성 + 실제 이벤트로 리허설
- **산출물**: 데모 영상 + 발표 자료 + 실사용 이벤트 데이터
